## Scope

- `src/phonology/lsj_extractor.py`: 1731 → 372 lines. Now a shim that owns
  - the `_pos_overrides` cache, `_empty_pos_overrides`, and `_load_pos_overrides`
    (so `monkeypatch.setattr(lsj_extractor_module, "_pos_overrides", None)`,
    `monkeypatch.setattr(..., "resolve_repo_data_dir", ...)`, and
    `monkeypatch.setattr(..., "transliterate", ...)` continue to take effect);
  - `main` and `run_cli` (so
    `monkeypatch.setattr(lsj_extractor_module, "main", fake_main)` is honoured
    by `run_cli` on the same module).
- New `src/phonology/lsj/` sub-package:
  - `__init__.py` (125): facade re-exporting the public surface.
  - `_constants.py` (150): POS / gender / dialect maps, regex patterns, frozen
    sets.
  - `_xml.py` (92): XML traversal helpers (`_elem_text`, `_find_*`,
    `_local_name`).
  - `_normalize.py` (105): Beta Code and headword normalisation, surface-form
    heuristics. `_looks_like_known_numeral_key` late-imports
    `_load_pos_overrides` from the shim so test resets propagate.
  - `_intro.py` (141): heading- and sense-intro text gathering, inline POS
    candidate enumeration.
  - `_heading.py` (610): heading context analysis, dialect-decision predicates,
    `_leading_dialect_labels` and friends.
  - `_fields.py` (290): headword extraction, POS / gender / gloss / dialect
    inference.
  - `_extract.py` (148): `extract_entry`. Resolves `_load_pos_overrides`,
    `to_ipa`, and `transliterate` from the shim at call time so existing
    monkeypatches still apply.
  - `_xml_iter.py` (87): `iter_xml_entries`, `find_xml_files`, `extract_all`.
  - `_document.py` (106): `_document_dialect*`, `build_lexicon_document`,
    `validate_document`.
- `src/phonology/build_lexicon.py`: `_FINGERPRINT_INPUTS` now lists every file
  in the new sub-package alongside the shim so the lexicon fingerprint picks
  up any source change to LSJ extraction.

All new modules pin their logger to `"phonology.lsj_extractor"` so existing
`caplog.set_level(..., logger="phonology.lsj_extractor")` blocks keep
capturing diagnostics from any sub-module.

## Compatibility Notes

- The shim deliberately re-implements `main` and `run_cli` rather than
  delegating to the sub-package. This keeps `extract_all`,
  `build_lexicon_document`, `validate_document`, and `_load_pos_overrides`
  resolution under the shim's namespace, where test monkeypatches reside.
- `_extract.extract_entry` resolves `_load_pos_overrides`, `to_ipa`, and
  `transliterate` lazily through the shim for the same reason.
- `_FINGERPRINT_INPUTS` now tracks the full sub-package; if a future
  contributor adds another `lsj/_*.py` file, it must be appended there too so
  the cached lexicon regenerates when that file changes.
